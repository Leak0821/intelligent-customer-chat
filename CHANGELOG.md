## 2026-05-26

### 分支

- `feat/lock-runtime-stack`

### 修改点

- 为回复发送链路补充 `send_readiness`、发送审批和派发记录持久化骨架
- 新增 `dispatch` 管理接口，支持查看发送记录与显式放行发送
- 增加开发踩坑文档，记录上下文压缩、正则识别、测试隔离等真实问题
- 将发件适配层从纯 `noop` 扩展为可切换 `noop / smtp`
- 增加 SMTP 配置项、消息构造测试和发送失败后的状态保持测试
- 为派发链路补充重试状态、退避策略、补偿服务，以及本地/XXL-JOB 重试入口
- 新增手工重试与批量补偿接口，并补充重试成功、重试耗尽的测试覆盖
- 新增 `review_records` 审核留痕、发送批准/驳回接口，以及回放中的审核轨迹
- 为派发记录补充最新触发来源、触发人、触发原因，增强发送审计信息
- 为草稿补充 `draft_version`、`last_edited_by`，支持驳回后修订并再次送审
- 新增修订草稿接口与对应测试，打通“拒绝 -> 修订 -> 再次审核”闭环
- 将售后业务查询适配层接到本地业务数据目录，不再只依赖硬编码返回
- 新增业务数据管理接口，可运行时维护订单、物流、售后策略样本
- 收口部分状态判断到领域对象，减少业务层散落的 `getStatus() == ...` 写法
- 新增工作流最小评估样本服务与接口，便于按 `runId` 或最近记录做复盘
- 评估样本会汇总意图、路由、业务事实、知识检索、草稿状态、审核结论和发送风险标记

### 结果

- 第一阶段已经具备“生成草稿 -> 人工审核 -> 草稿修订 -> 再次送审 -> 派发记录 -> 安全 no-op / 可选 SMTP -> 失败重试补偿”的发送闭环骨架
- 当前批次已通过 `42` 个测试
- 当前分支已持续推送到远端
