# 研究区

`research/` 用于保存正式规格形成前的探索材料。

典型内容包括：

- 需求澄清
- 方案对比
- 风险记录
- 领域问题
- 初期数据或流程观察

规则：

- `research` 可以不完整
- `research` 允许中途调整方向
- `research` 不是实现合同
- 已确认的结论应提升到 `openspec/changes/`
