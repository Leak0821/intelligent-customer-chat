## 建议实现切片

### 切片 1：邮件接入与基础主链路骨架

- 建立 `IMAP` 接入 adapter 与去重、重试、已读控制基础能力
- 建立 `Java + Spring Boot + 显式状态主链路` 主流程骨架
- 打通邮件接入、清洗、状态推进、事件记录的最小空链路

参考编码计划：

- `research/first-stage-slice-1-coding-plan.md`

### 切片 2：意图归一化与路由

- 实现意图重写 / 归一化结构化输出
- 实现一级场景与首批子意图识别
- 实现主问题、附带问题、缺失实体判断

### 切片 3：上下文加载与业务数据接入

- 实现历史邮件线程摘要加载
- 实现订单、物流、售后规则 gateway / adapter 抽象
- 打通至少一个可用的订单或物流查询链路

### 切片 4：轻量 RAG 与知识补充

- 建立首批知识源整理方式
- 实现轻量检索与结构化检索结果输出
- 实现 facts 与 knowledge 的组装顺序

### 切片 5：售前与售后首批子意图闭环

- 打通售前 4 个首批子意图的草稿生成链路
- 打通售后 4 个首批子意图的草稿生成链路
- 实现“直接草稿 / 先追问 / 转人工”三类输出

### 切片 6：审核、兜底与状态模型

- 实现审核与兜底决策层
- 落地 `draft_ready / follow_up_needed / human_review_required / blocked`
- 落地发送前状态与下一步动作控制

### 切片 7：日志、回放与最小评估

- 为关键节点落结构化事件
- 支持按 `message_id` 或 `run_id` 回放链路
- 沉淀最小评估样本与错误案例复盘基础
